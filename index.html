Sobre mi

Soy egresado de la Licenciatura en Computación por la Universidad Autónoma Metropolitana (UAM) y me apasiona el desarrollo Back-End. He realizado proyectos con frameworks como Spring Boot y Laravel, aunque tambien tengo conocimientos básicos con herramientas para el desarrollo de Front-End como Html, Css, Angular y Javascript. Mi enfoque está en la optimización del rendimiento, la integración de herramiestas de seguridad y autenticación, conexión de bases de datos y el desarrollo de APIs REST Full, buscando siempre aportar valor con soluciones innovadoras.

Proyectos

Gastos

Este proyecto fue desarrollado en Spring Boot como parte de un curso sobre esta tecnología. El proyecto permite listar tanto los gastos fijos del mes, asi como los gastos generados durante el mes en curso, este cuenta principalmente con dos roles administrador y usuario. El proyecto incluye un sistema de autenticación de usuarios (login) con Spring Security y JSON Web Token (JWT), cifrado de contraseñas, operaciones CRUD y la gestión de roles de usuario para diferentes niveles de acceso.

GYM (EN PROCESO)

Este proyecto consiste en el desarrollo de una plataforma web para la gestión de un gimnasio, organizada en dos roles principales: el administrador, encargado de gestionar tanto al personal como a los usuarios, y el personal, que se ocupa del registro de los usuarios. El objetivo es automatizar procesos clave, como el manejo de pagos, la gestión de horarios y la integración de PayPal como método de pago mediante tarjeta. Además, se implementa un sistema de roles con autenticación y autorización, utilizando Spring Security y JWT para garantizar la seguridad y el acceso adecuado a las funcionalidades del sistema.

Sistema de Prestamos

El proyecto fue realizado dentro de la Universidad utilizando el framework Laravel. EL objetivo es gestionar de manera eficiente los préstamos de equipos, como laptops y proyectores, tanto para alumnos como para profesores, la reservación de salas para eventos universitarios y un apartado de soporte, para dar asistencia a los equipos de los docentes y personal del edificio. En el proyecto se implementaron funcionalidades, como la gestión de roles de usuarios, autenticación y control de acceso, manejo de errores, filtrado de datos, automatización de procesos y conexiones a bases remotas.

Mis Habilidades

Laravel

En este framework, he desarrollado una API REST utilizando el patrón de arquitectura MVC, implementado la conexión a bases de datos SQL tanto locales como remotas, y gestionado la autenticación de usuarios. Además, he implementado roles de usuario, funciones CRUD y automatizado tareas, como la funcionalidad de suspensiones automáticas.

Spring Boot

En Spring Boot, he desarrollado una API REST, configurado la conexión a bases de datos SQL y NoSQL, y gestionado la autenticación de usuarios. Además, he implementado Spring Security, configurado roles de usuario y desarrollado operaciones CRUD.

API REST

He diseñado e implementado una API RESTful utilizando Spring Boot y Laravel. La API está estructurada para interactuar con los recursos mediante endpoints, utilizando métodos HTTP estándar (GET, POST, PUT, DELETE) y gestionando códigos HTTP de respuesta adecuados. Además, he implementado un enfoque stateless, donde cada solicitud es independiente, y he utilizado JSON como formato de respuesta para garantizar una comunicación eficiente y flexible.

Bases de datos

He implementado consultas para optimizar el rendimiento, utilizando índices y técnicas de normalización. He modelado bases de datos relacionales utilizando MySQL, implementando tablas, relaciones y claves foráneas y primarias para garantizar una estructura de datos sólida y optimizada. Además, he trabajado con bases de datos no relacionales como MongoDB.

Seguridad

En los proyectos que he realizado, he integrado JSON Web Token (JWT) para la autenticación de usuarios. Además, he implementado técnicas de encriptación de contraseñas utilizando herramientas como bcrypt, asegurando que los datos sensibles estén protegidos en todo momento. También he manejado los permisos de roles para controlar el acceso a diferentes partes de las páginas, creando un sistema flexible y escalable que asigna diferentes privilegios según las necesidades del proyecto.

Control de Versiones

He utilizado GitHub como sistema de control de versiones para gestionar mis proyectos. A través de esta plataforma, subo mis proyectos, realizo cambios y llevo un registro detallado de las modificaciones mediante commits y pull requests. Esto me ha permitido trabajar de manera colaborativa, mantener un historial de versiones y gestionar de manera eficiente el desarrollo de mis proyectos.

Metodologias

He implementado metodologías agiles como Scrum y GitFlow. Utilice Scrum para gestionar proyectos en equipo mediante sprints, lo que permitió una mejora continua a lo largo del proyecto. Al mismo tiempo, implementé GitFlow para organizar el control de versiones, utilizando ramas específicas como feature, develop y master para gestionar el código y facilitar la colaboración entre los integrantes, evitando conflictos y asegurando un flujo de trabajo estructurado, lo que hizo que la entrega de tareas fueran mas rapidas y eficientes.
